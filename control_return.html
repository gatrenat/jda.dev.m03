El programa s'ha acabat

Considera el següent programa:

 1/*
 2 * Programa que demana un text i un nombre de cops, i mostra el text
 3 * repetit tants cops com se li indiqui
 4 */
 5public class RepeteixText {
 6    public static void main(String[] args) {
 7        System.out.println("Text?");
 8        String text = Entrada.readLine();
 9        System.out.println("Cops?");
10        int cops = Integer.parseInt(Entrada.readLine());
11        if (text.isEmpty()) {
12            System.out.println("Error: el text no pot ser buit");
13        } else if (cops < 1) {
14            System.out.println("Error: com a mínim un cop");
15        } else {
16            for (int i=0; i<cops; i++) {
17                System.out.println(text);
18            }
19        }
20    }
21}

El programa és molt simple. Es limita a:

  • demanar els dos valors que necessita

  • comprovar que els valors introduïts siguin correctes per

  • mostrar un error si els valors no són adequats

  • repetir un text un nombre de cops, si els valors resulten adequats.

El programa és correcte, però… El però està en que si ja saps que el text no es correcte, podries estalviar els usuaris d'introduir el nombre de cops, no creus?

No és problema resoldre-ho, oi? Tenim if!

 1/*
 2 * Programa que demana un text i un nombre de cops, i mostra el text
 3 * repetit tants cops com se li indiqui
 4 */
 5public class RepeteixText {
 6    public static void main(String[] args) {
 7        System.out.println("Text?");
 8        String text = Entrada.readLine();
 9        if (text.isEmpty()) {
10            System.out.println("Error: el text no pot ser buit");
11        } else {
12            System.out.println("Cops?");
13            int cops = Integer.parseInt(Entrada.readLine());
14            if (cops < 1) {
15                System.out.println("Error: com a mínim un cop");
16            } else {
17                for (int i=0; i<cops; i++) {
18                    System.out.println(text);
19                }
20            }
21        }
22    }
23}

Aquesta nova versió només pregunta el nombre de cops si el text és adequat.

T'has fixat, però, quants nivells d'indentació han calgut per poder anar salvant els diferents casos d'error?

És una llàstima perquè, un cop ja sabem que el text està buit, ja podríem finalitzar el programa, sense necessitat de fer res més. Hi haurà alguna manera d'indicar en Java que el programa s'acaba en un determinat moment?

La resposta és sí. En qualsevol moment podem retornar de l'execució del nostre programa fent servir la paraula reservada return.

Mira com queda el programa:

 1/*
 2 * Programa que demana un text i un nombre de cops, i mostra el text
 3 * repetit tants cops com se li indiqui
 4 */
 5public class RepeteixText {
 6    public static void main(String[] args) {
 7        System.out.println("Text?");
 8        String text = Entrada.readLine();
 9        if (text.isEmpty()) {
10            System.out.println("Error: el text no pot ser buit");
11            return;
12        }
13        System.out.println("Cops?");
14        int cops = Integer.parseInt(Entrada.readLine());
15        if (cops < 1) {
16            System.out.println("Error: com a mínim un cop");
17            return;
18        }
19        // Ara sabem que les dades d'entrada són vàlides
20        for (int i=0; i<cops; i++) {
21            System.out.println(text);
22        }
23    }
24}

Potser no hem guanyat molt en nombre de línies, però sí en complexitat d'indentació. Si l'estructura del nostre programa és més simple, hem guanyat en claredat.

Fixa't com, si el programa executa la línia 13, sabem segur que el text no és buit. Altrament hauria finalitzat l'execució a la línia 11!

Així, en arribar a la línia 19, sabem segur que tant el text com el nombre de cops tenen els valors esperats.

Ho podem dibuixar de la següent manera:

Podem imaginar-nos cada comprovació d'error com una mena de vigilant que no deixa passar les execucions que no compleixen les condicions requerides. En el cas de l'exemple, aquestes condicions són que el text no pot ser buit ni el nombre de cops inferior a 1.
