Exercici 35_18. EnvoltaString és compara amb Strings

Context

  • Carpeta de lliurament: 35_18_envoltastring_str/

  • Continguts relacionats: Clau-Valor

  • Com lliurar-lo: instruccions

  • [✓] Exercici amb autoavaluació

Enunciat

Crea una nova versió de la classe EnvoltaString a partir de la versió de l'exercici anterior de manera que ara també permeti comparar un EnvoltaString directament amb un String.

Considera aquest UsaEnvoltaString:

 1 public class UsaEnvoltaString {
 2     public static void main(String[] args) {
 3         String nom1 = "Renat";
 4         String nom2 = args.length > 0 ? args[0] : "nom per defecte";
 5         System.out.printf("\"%s\".equals(\"%s\") -> %b%n", nom1, nom2, nom1.equals(nom2));
 6
 7         EnvoltaString envoltaNom1 = new EnvoltaString(nom1);
 8         EnvoltaString envoltaNom2 = new EnvoltaString(nom2);
 9         System.out.printf("%s.equals(%s) -> %b%n", envoltaNom1, envoltaNom2, envoltaNom1.equals(envoltaNom2));
10         System.out.printf("%s.equals(\"%s\") -> %b%n", envoltaNom1, nom2, envoltaNom1.equals(nom2));
11         System.out.printf("\"%s\".equals(%s) -> %b%n", nom1, envoltaNom2, nom1.equals(envoltaNom2));
12     }
13 }

Que presentarà la sortida:

 $ java UsaEnvoltaString RENAT
 "Renat".equals("RENAT") -> false
 ENVOLTAT("Renat").equals(ENVOLTAT("RENAT")) -> true
 ENVOLTAT("Renat").equals("RENAT") -> true
 "Renat".equals(ENVOLTAT("RENAT")) -> false

Nota: fixa't en la darrera comparació de UsaEnvoltaString. Creus que mai podrà ser true? Perquè? Posa la teva resposta en els comentaris del teu codi.
