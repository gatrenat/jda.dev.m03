Breu consideració sobre eficiència

Comparem un altre cop les darreres versions del nostre ComptaAiEs

La versió que crida a quantesOcurrencies() cada cop que necessita el valor, sense importar-li si ja s'havia calculat abans:

 1/* programa que mostra quantes 'a's i 'e's té el text llegit per stdin */
 2public class ComptaAiEs {
 3    public static void main(String[] args) {
 4        System.out.println("Introdueix un text");
 5        String entrada = Entrada.readLine();
 6        comptaLletra(entrada, 'a');
 7        comptaLletra(entrada, 'e');
 8        comparaAiEs(entrada);
 9    }
10    public static void comptaLletra(String text, char lletra) {
11        int comptador = quantesOcurrencies(text, lletra);
12        System.out.println("Nombre de '" + lletra + "'s: " + comptador);
13    }
14    public static void comparaAiEs(String text) {
15        int comptadorA = quantesOcurrencies(text, 'a');
16        int comptadorE = quantesOcurrencies(text, 'e');
17        if (comptadorA > comptadorE) {
18            System.out.println("Hi ha més 'a's que 'e's");
19        } else if (comptadorA < comptadorE) {
20            System.out.println("Hi ha menys 'a's que 'e's");
21        } else {
22            System.out.println("Hi ha tantes 'a's com 'e's");
23        }
24    }
25    public static int quantesOcurrencies(String text, char lletra) {
26        int comptador = 0;
27        for (int i=0; i < text.length(); i++) {
28            if (text.charAt(i) == lletra) {
29                comptador += 1;
30            }
31        }
32        return comptador;
33    }
34}

En total, en aquest programa es crida dos cops quantesOcurrencies() per cada vocal, i sempre pel mateix text. El cert és que, un cop ja has comptat les 'a' que conté un text, per molt que les tornis a comptar, seguiran havent-hi les mateixes. Malauradament quantesOcurrencies() no sap que ja les ha comptades abans i torna a fer el càlcul un altre cop per obtenir el mateix resultat.

Et proposo la següent versió que cridaria quantesOcurrencies() el mínim possible:

 1/* programa que mostra quantes 'a's i 'e's té el text llegit per stdin */
 2public class ComptaAiEs {
 3    public static void main(String[] args) {
 4        System.out.println("Introdueix un text");
 5        String entrada = Entrada.readLine();
 6        int numAs = quantesOcurrencies(entrada, 'a');
 7        int numEs = quantesOcurrencies(entrada, 'e');
 8        mostraOcurrencies('a', numAs);
 9        mostraOcurrencies('e', numEs);
10        mostraComparacio('a', numAs, 'e', numEs);
11    }
12    public static void mostraOcurrencies(char lletra, int quantes) {
13        System.out.println("Nombre de '" + lletra + "'s: " + quantes);
14    }
15    public static void mostraComparacio(char lletra1, int quantes1, char lletra2, int quantes2) {
16        if (quantes1 > quantes2) {
17            System.out.println("Hi ha més '" + lletra1 +"'s que '"+lletra2+"'s");
18        } else if (quantes1 < quantes2 ) {
19            System.out.println("Hi ha menys '" + lletra1 +"'s que '"+lletra2+"'s");
20        } else {
21            System.out.println("Hi ha tantes '" + lletra1 +"'s com '"+lletra2+"'s");
22        }
23    }
24    public static int quantesOcurrencies(String text, char lletra) {
25        int comptador = 0;
26        for (int i=0; i < text.length(); i++) {
27            if (text.charAt(i) == lletra) {
28                comptador += 1;
29            }
30        }
31        return comptador;
32    }
33}

Fixa't que aquesta versió només calcula un cop les ocurrències de cada lletra, i guarda els resultats en les variables temporals numAs numEs, per després passar els valors als procediments mostraOcurrencies() i mostraComparacio().

Podríem dir que aquesta versió és més eficient que l'anterior, ja que no torna a realitzar càlculs innecessàriament.

Abans que ens emocionem massa, cal que tinguem present el cost que representa aquesta optimització:

  • ens cal guardar els valors en variables temporals. En aquest cas numAs i numEs

  • la signatura dels nostres mòduls s'ha complicat. Ara requereixen més paràmetres

  • l'ús d'aquests mòduls modificats també s'ha complicat: ara requereixen comptar les ocurrències abans de ser usats.

Si bé pot resultar molt temptador optimitzar els nostres codis, sovint afegir més complexitat no és una bona idea.

Així doncs, què fem? Optimitzem o no optimitzem?

Com a regla general, evitarem optimitzar massa aviat, especialment si és a costa de complicar el codi. Un cop finalitzat el programa, i només si la seva execució resulta massa lenta, passarem a optimitzar tot mirant de minimitzar la pèrdua de simplicitat.

Vols saber-ne més?

Quan desenvolupem hem de tenir sempre en ment la famosa frase de Donald Knuth Premature Optimization Is the Root of All Evil (l'optimització prematura és l'origen de tots els mals)

L'optimització prematura no és només una cosa de desenvolupadors. Per exemple, aquest article ho explica des d'un punt de vista més general.
