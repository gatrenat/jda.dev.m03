Les variables abans dels mòduls

Fins ara no ens hem preocupat gaire d'una característica important de les variables: on tenen validesa. En paraules més tècniques, el seu àmbit o scope.

Sabem que una variable està disponible per ser assignada a partir de que s'executa la seva definició, i que pot ser consultat el seu valor a partir de la seva inicialització

 1// Aquí la variable llenguatge és desconeguda. Qualsevol al·lusió a 'llenguatge' que
 2// no sigui per declarar-la, donarà error
 3
 4String llenguatge;     // declaració
 5
 6// Ara ja puc assignar-li un valor. ex. puc fer llenguatge = "java";
 7
 8// Encara no puc consultar el seu valor. ex. donarà error System.out.println(llenguatge);
 9
10// Ja no puc tornar a declarar una variable amb aquest mateix nom
11
12llenguatge = "java";   // inicialització
13
14// Ara puc encara assignar-li un nou valor. ex. llenguatge = "javascript";
15// També puc consultar el seu valor. ex. System.out.println(llenguatge);

També hem pogut experimentar que si declarem una variable dins del bloc d'un condicional o un bucle, aquesta variable només pot ser consultada dins del bloc:

 1// Aquí encara no podem parlar de la variable llenguatge, a banda de declarar-la
 2// Si la declarem, però, tornar-la a declarar dins del bucle provocaria un error
 3
 4if (true) {
 5
 6    // Aquí encara no existeix la variable llenguatge
 7
 8    String llenguatge = "java";     // declaració i inicialització
 9
10    // Ara ja està disponible la variable per assignar i consultar
11
12    System.out.println(llenguatge); // podem consultar-la sense problemes
13
14}
15
16// A partir d'aquí, la variable ja no existeix. La podríem declarar sense problemes
17int llenguatge = 42;
18System.out.println(llenguatge);

En el cas dels bucles for vam experimentar un cas particular quan declaràvem la variable de recorregut dins dels parèntesis del bucle:

 1// La variable linia no existeix encara
 2// Si la declarem aquí, al for donarà error
 3
 4for (int linia=0; linia<10; linia++) {  // a partir d'aquí, linia ja existeix
 5
 6    // podem usar linia dins del bloc del for
 7    System.out.println(linia);
 8}
 9
10// A partir d'aquí linia no existeix. La podem tornar a declarar
11String linia = Entrada.readLine();
