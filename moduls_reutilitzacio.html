Reutilització dels mòduls

Ara ja tenim una visió prou completa dels mòduls i sabem que els que ens ofereix Java (ex. length() dels Strings) els podem utilitzar en tots els nostres programes.

Com podríem fer el mateix amb els nostres mòduls? És a dir, com podem reutilitzar un mòdul que hem definit nosaltres en diferents programes?

Posem un exemple: un cop tenim desenvolupat el programa Autodestruccio que varem conèixer aquí, la funció respostaABoolean() ens pot ser útil per altres programes que requereixin demanar confirmació dels usuaris. Suposem que hem de fer una nova versió de Autodestruccio anomenat AutodestruccioTotal que permeti destruir, no tant sols l'equip on s'executa sinó tot el món (!).

Una primera resposta seria copiant i enganxant el mòdul al nou programa. Per exemple:

 1/*
 2 * Programa que destrueix el món.
 3 * Abans de fer res, demana confirmació als usuaris.
 4 * Nota: per si algun usuari acaba confirmant, ha estat un plaer
 5 * conèixer-te
 6 */
 7public class AutodestruccioTotal {
 8    public static void main(String[] args) {
 9        System.out.println("Estic apunt de destruir el món. " +
10                           "Estàs d'acord?");
11        boolean confirma = true;
12        while (confirma) {
13            String resposta = Entrada.readLine();
14            confirma = respostaABoolean(resposta);
15            if (confirma) {
16                System.out.println("Mira que és una decisió definitiva. " +
17                                   "Segur que vols?");
18            }
19        }
20        System.out.println(
21                "Sàvia resposta. " +
22                "De totes maneres la destrucció del món és una tasca que " +
23                "ja estem fent entre tots!");
24    }
25
26    /*
27     * Donada una resposta textual, aquesta funció tradueix la resposta a
28     * un booleà.
29     * Considera true quan la resposta és, independentment de majúscules i
30     * sense considerar espais a l'inici ni al final,
31     * "sí", "s", "yes" o "y", i algunes variants amb errors ortogràfics.
32     * Altrament considera false.
33     */
34    public static boolean respostaABoolean(String resposta) {
35        if (null == resposta) {     // considerem false la resposta null
36            return false;
37        }
38        resposta = resposta.toLowerCase();
39        if (resposta.equals("s") || resposta.equals("y")) {
40            return true;
41        }
42        if (resposta.equals("sí") || resposta.equals("yes")) {
43            return true;
44        }
45        if (resposta.equals("si") || resposta.equals("vale") || resposta.equals("yeah")) {
46            return true;
47        }
48        return false;
49    }
50}

Aquesta solució ja ens estalvia feina, sí, però no és del tot satisfactòria. Ja sabem que la duplicació de codi és quelcom a defugir.

Java ens ho posa molt fàcil.

Considera la nova versió de AutodestruccioTotal:

 1/*
 2 * Programa que destrueix el món.
 3 * Abans de fer res, demana confirmació als usuaris.
 4 * Nota: per si algun usuari acaba confirmant, ha estat un plaer
 5 * conèixer-te
 6 */
 7public class AutodestruccioTotal {
 8    public static void main(String[] args) {
 9        System.out.println("Estic apunt de destruir el món. Estàs d'acord?");
10        boolean confirma = true;
11        while (confirma) {
12            String resposta = Entrada.readLine();
13            confirma = Autodestruccio.respostaABoolean(resposta);
14            if (confirma) {
15                System.out.println("Mira que és una decisió definitiva. " +
16                                   "Segur que vols?");
17            }
18        }
19        System.out.println(
20                "Sàvia resposta. " +
21                "De totes maneres la destrucció del món és una tasca que " +
22                "ja estem fent entre tots!");
23    }
24}

Si et fixes, la línia 13 fa el truc: simplement indiquem que respostaABoolean() es troba dins del programa Autodestruccio i ja està!

Amb la configuració que tens del teu repositori local, hauria de funcionar-te simplement col·locant AutodestruccioTotal.java i Autodestruccio.java a la mateixa carpeta,

$ ls Autodestruccio*.java
Autodestruccio.java  AutodestruccioTotal.java
$ javac Autodestruccio*.java
$ java AutodestruccioTotal

Ara que ja tenim més d'un fitxer en joc, podrem trobar situacions en les que java no disposi de la darrera versió compilada de tots els fitxers. Per exemple, imagina que compiles AutodestruccioTotal.java. javac farà la compilació de Autodestruccio.java si aquest ha canviat. Tot correcte, però, si en aquest moment modifiquem Autodestruccio.java i no el recompilem, en intentar executar AutodestruccioTotal, es farà servir la versió anterior de Autodestruccio.class, sense els nous canvis. Per evitar sorpreses, cada cop que modifiquis un fitxer, et recomano que, de moment, facis com a l'exemple i indiquis a javac tots els fitxers font a compilar. Sovint amb javac *.java ja faràs si tot el codi que tens a la carpeta és vàlid.

Com troben javac i java els altres programes? Tot plegat funciona gràcies a la variable d'entorn anomenada CLASSPATH i les paraules public i static. De moment no ens preocuparem amb aquest temes.
