Un nou tipus de mòdul

Recuperem el codi tal i com l'havíem deixat:

 1/* programa que mostra quantes 'a's i 'e's té el text llegit per stdin */
 2public class ComptaAiEs {
 3    public static void main(String[] args) {
 4        System.out.println("Introdueix un text");
 5        String entrada = Entrada.readLine();
 6        comptaLletra(entrada, 'a');
 7        comptaLletra(entrada, 'e');
 8        comparaAiEs(entrada);
 9    }
10    public static void comptaLletra(String text, char lletra) {
11        int comptador = 0;
12        for (int i=0; i < text.length(); i++) {
13            if (text.charAt(i) == lletra) {
14                comptador += 1;
15            }
16        }
17        System.out.println("Nombre de '" + lletra + "'s: " + comptador);
18    }
19    public static void comparaAiEs(String text) {
20        int comptadorA = 0;
21        int comptadorE = 0;
22        for (int i=0; i < text.length(); i++) {
23            if (text.charAt(i) == 'a') {
24                comptadorA += 1;
25            } else if (text.charAt(i) == 'e') {
26                comptadorE += 1;
27            }
28        }
29        if (comptadorA > comptadorE) {
30            System.out.println("Hi ha més 'a's que 'e's");
31        } else if (comptadorA < comptadorE) {
32            System.out.println("Hi ha menys 'a's que 'e's");
33        } else {
34            System.out.println("Hi ha tantes 'a's com 'e's");
35        }
36    }
37}

Aquest codi ens permet comptar i comparar les ocurrències de les vocals 'a' i 'e' en un text donat.

Si bé fa la feina, el codi potser no ens agrada del tot. Si més no, el mòdul comparaAiEs() ha de recomptar les ocurrències de les dues lletres per poder fer la seva feina.

Si poguéssim aconseguir guardar d'alguna manera el nombre de vegades que apareixen les vocals 'a' i 'e', la comparació seria immediata, no?

Considera la següent versió amb molta atenció, que ara sí hi ha novetats:

 1/* programa que mostra quantes 'a's i 'e's té el text llegit per stdin */
 2public class ComptaAiEs {
 3    public static void main(String[] args) {
 4        System.out.println("Introdueix un text");
 5        String entrada = Entrada.readLine();
 6        comptaLletra(entrada, 'a');
 7        comptaLletra(entrada, 'e');
 8        comparaAiEs(entrada);
 9    }
10    public static void comptaLletra(String text, char lletra) {
11        int comptador = quantesOcurrencies(text, lletra);
12        System.out.println("Nombre de '" + lletra + "'s: " + comptador);
13    }
14    public static void comparaAiEs(String text) {
15        int comptadorA = quantesOcurrencies(text, 'a');
16        int comptadorE = quantesOcurrencies(text, 'e');
17        if (comptadorA > comptadorE) {
18            System.out.println("Hi ha més 'a's que 'e's");
19        } else if (comptadorA < comptadorE) {
20            System.out.println("Hi ha menys 'a's que 'e's");
21        } else {
22            System.out.println("Hi ha tantes 'a's com 'e's");
23        }
24    }
25    public static int quantesOcurrencies(String text, char lletra) {
26        int comptador = 0;
27        for (int i=0; i < text.length(); i++) {
28            if (text.charAt(i) == lletra) {
29                comptador += 1;
30            }
31        }
32        return comptador;
33    }
34}

A veure què ha passat aquí!

En primer lloc, ha aparegut un nou mòdul anomenat quantesOcurrencies() Aquest mòdul rep el text i una lletra i calcula el nombre d'ocurrències de la lletra dins del text. Això és exactament el que feia comptaLletra(), excepte per un petit detall: en comptes de mostrar el resultat per la sortida estàndard, quantesOcurrencies() el retorna!

Ara, a cada lloc on abans havíem d'incloure un bucle per comptar quantes vegades apareix una lletra al text, hem substituït el bucle per una crida a quantesOcurrencies(). Mira les línies 11, 15 i 16.

Fixa't en les diferències de quantesOcurrencies() respecte a altres mòduls com ara comptaLletra():

  • a la declaració del mòdul apareix, en comptes de void, un tipus. En aquest cas és int, però podria ser qualsevol altre.

    Sovint ens referirem a aquest tipus com el tipus de retorn del mòdul.

  • la darrera línia del mòdul conté return comptador;. El comptador és una variable de tipus enter, a l'igual que el tipus de retorn del mòdul.

    En realitat no cal que sigui una variable. Pot ser qualsevol expressió que retorni un valor del mateix tipus que el tipus de retorn.

  • la manera de cridar a quantesOcurrencies() també és diferent. En comptes de fer la crida com a instrucció (fixa't en la línia 6 per exemple) ho fem com a expressió. Per exemple, la línia 11 assigna el valor resultant de cridar a quantesOcurrencies() a la variable comptador.

  • la manera d'anomenar aquest mòdul és diferent la que hem vist fins ara. La recomanació amb els mòduls anteriors era fer servir un nom d'acció per descriure l'acció que realitzava. En aquest cas "quantes ocurrències" no és ben bé un verb sinó un substantiu similar al que faríem servir per anomenar una variable!

Amb totes aquestes diferències ja pots imaginar que estem davant d'una bestiola diferent de les anteriors. I així és.

A aquest tipus de mòduls que retornen un valor, els coneixem com funcions, en contraposició als que no retornen cap valor (void) que es solen anomenar procediments.

Val a dir que molta gent parla directament de funcions per referir-se als mòduls siguin procediments o funcions. Això és especialment freqüent en llenguatges com Java, que no ofereixen distinció sintàctica entre els dos tipus de mòduls. La distinció és, però, prou rellevant com per que altres llenguatges ho remarquin. Per exemple, els llenguatges Pascal i Ada fan servir Procedure/procedure i Function/function per declarar mòduls d'un o altre tipus.

De moment nosaltres distingirem els dos tipus de mòdul. Ens resultarà fàcil si veiem que:

  • Si la signatura declara void i no hi ha valor darrera del return, és un procediment.

  • Si la signatura declara un tipus (no void) i darrera del return hi ha un valor d'aquest tipus, és una funció.

Escollirem procediments quan vulguem que es realitzi alguna acció i funcions quan ens interessi obtenir un valor resultant.
